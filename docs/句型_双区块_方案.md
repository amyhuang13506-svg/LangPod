# 句型 Tab 改版方案：日常社交 + 商务英语双区块

> 状态：**待确认**。确认后按「分期执行」动代码。
> 日期：2026-07-17
> 对标：`vocab_v2_双区块_方案.md`（词汇 tab 已落地的双区块模式）

## 1. 背景与目标

**现状：**
- 句型 tab 一级导航是 6 个并列 chips：今天 / 日常 / 玩梗 / 观点 / 会话技巧 / 进阶地道
- 共 34 分类、约 460 条（291 条精选库 + 169 条今天句型）
- 内容 100% 偏日常社交，**完全没有商务/职场线**（仅「进阶地道」下有 1 个「职场表达」分类 9 条）
- 6 个 chips 已经有点挤，再加内容线无处可放

**目标：**
1. 句型 tab 改为「**日常社交 | 商务英语**」左右双区块，与词汇 tab 的「日常词汇 | 生活场景」同构
2. 现有 6 个 chips 原样收进「日常社交」区块，**该区块零改动**
3. 新增「商务英语」内容线：面向职场场景的实用句型

## 2. 信息架构与 UI

```
┌──────────────────────────────────────┐
│              句型            [我的]   │
│                                      │
│   ┌────────────┐  ┌────────────┐     │

│                                      │
│   ── 选中「日常社交」──                 │
│   今天 · 日常 · 玩梗 · 观点 · 会话技巧 · 进阶地道 │  ← 现有 chips 原样
│   → 现有分类区块横滑卡片，完全不动        │
│                                      │
│   ── 选中「商务英语」──                 │
│   会议 · 邮件 · 谈判 · 面试 · 汇报 · 职场社交  │  ← 新的 group chips
│   → 同构：分类区块 + 横滑卡片 + 查看更多   │
└──────────────────────────────────────┘
```

**设计细节：**
- 大区块切换：两个等宽胶囊 + icon，完全复制 `VocabularyView.sectionSwitcher` 的样式（`bubble.left.and.bubble.right.fill` / `briefcase.fill`）
- 日常社交区 = 现状原样：group chips + categorySection 横滑 + 查看更多网格 + 翻页详情，**零改动**
- 商务英语区 = 同一套组件复用（`ExpressionSceneCard` / `ExpressionCategoryAllView` / `ExpressionPagerView`），只是数据来自 business 组

## 3. 商务英语内容目录 v1（待确认）

按「职场里真正开口的时刻」组织，每分类 9-10 条，结构与现有句型完全一致（english + meaning_zh + usage_zh + 3 例句 + A/B 场景对话 + 封面）：

| group id | 中文 chip | 分类（首批粗体） |
|---|---|---|
| biz_meeting | 会议 | **开场与主持**、**发言与插话**、表态同意反对、收尾与行动项 |
| biz_email | 邮件 | **开头与结尾**、**请求与跟进**、延期与道歉、进度汇报 |
| biz_negotiate | 谈判 | **报价与议价**、让步与条件、委婉拒绝、推进成交 |
| biz_interview | 面试 | **自我介绍**、**经历与优势**、反问面试官、薪资谈判 |
| biz_present | 汇报 | 开场与结构、讲数据、过渡与强调、Q&A 应对 |
| biz_social | 职场社交 | **Small talk**、networking 破冰、感谢与致意、告别与跟进 |

- **首批 8 分类**（加粗）≈ 75 条；全目录 24 分类 ≈ 230 条
- 成本：每条 1 封面 + 1 场景图 ≈ ¥1 → 首批 ~¥75、全量 ~¥230

**存量迁移建议（零成本白捡内容）：**
- 「进阶地道 · 职场表达」9 条（circle back / loop you in / EOD…）→ 移到 **商务英语 · 职场社交** 或单独成「职场黑话」分类
- 「进阶地道 · 正式与礼貌场合」9 条（I'd appreciate it if / at your earliest convenience…）→ 移到 **商务英语 · 邮件**
- 迁移只需改 catalog 的归属 + 重传 index，**内容/媒体全部复用不重跑**

## 4. 数据模型与 OSS 结构

**核心决定：不拆索引，给 group 加一个 `section` 字段。**

- `expressions/index.json` 的每个 group 增加 `"section": "social" | "business"`
- App 端 `ExpressionGroup` 加 `section` 字段，chips 按当前 section 过滤
- 分类 JSON 路径不变（`expressions/{category_id}.json`），**零接口新增**
- 老版本 App 兼容性：老版本不认 `section`，会把 business 组当普通 chip 平铺显示 —— 内容正常可看，只是没有双区块。⚠️ 若不接受，方案 B 是 business 走独立 `expressions/business_index.json`（老版本完全看不到）

**catalog 改动：**
- `expression_catalog.py` 的 GROUPS 每项加 `"section"`；现有 6 组标 `social`，新增 6 组标 `business`

## 6. Pipeline 改动清单

1. `expression_catalog.py`：GROUPS 加 `section` 字段；新增 6 个 business 组 + 分类目录；存量 2 分类迁移归属
2. `upload_expressions.py`：index 生成带上 `section`
3. `generate_expressions.py`：直接复用（business 分类靠 hint 描述职场语境，走同一条生成链）
4. `generate_expression_card_covers.py` / `generate_expression_scenes.py`：新增 business 组配色（建议**深蓝 / 石板灰**系，和日常社交的暖色系拉开）
5. 无需新脚本

## 7. App 改动清单

- `Expression.swift`：`ExpressionGroup` 加 `section: String?`（可选，兼容老数据）
- `ExpressionStore`：加 `section: ExpressionSection`（UserDefaults `patternSection`）+ `groupsInSection` 过滤；切区块时默认选中该区块第一个 group
- `PatternsTabView`：顶部加 `sectionSwitcher`（复制词汇 tab 样式）；groupChips 改为遍历 `groupsInSection`
- `Analytics` 新事件：`pattern_section_switch`（social / business）
- 空态：商务英语内容未上线前显示「商务英语即将上线」占位

## 8. 分期执行

- **P3（补齐）**：全目录 24 分类；商务场景的今天句型（若播客加商务选题）

## 9. 决策记录（7/17 已确认）

1. ✅ **大区块命名**：日常社交 / 商务英语
2. ✅ **商务目录**：第 3 节 6 组 24 分类，**全量上线**（不做首批分期）
3. ✅ **存量迁移**：职场表达 → 商务·职场社交；正式与礼貌场合 → 商务·邮件。移走后「进阶地道」剩 4 分类
4. ⏸ **免费策略**：本期不管，维持现状（今天·初级首条免费，其余 Pro），内容做完再定
5. ✅ **老版本兼容**：走 section 字段（方案 A）。老版本会把 business 当普通 chip 平铺显示，可接受
6. ✅ **商务区配色**：深蓝 / 石板灰系，与日常线暖色拉开

**⚠️ 关键密度决策（7/17 确认，与原方案不同）：**
- **每个分类 25 张卡**（不是原方案的 9-10），理由：整个页面要填满，不能稀
- 商务线总量：24 分类 × 25 ≈ **600 条**（含迁移的 18 条 → 新生成 ~582 条）
- 成本：~¥600、5-6 小时
- 日常线维持现有密度（9-10/分类）不动

**因此生成方式必须改：**
现有 `generate_expressions.py` 一次 GPT 出 8-12 条（validate 限 6-14）。25 条一次出会触发超长 JSON 截断（今天句型那次的教训）。
→ 新增「补足到 N 条」模式：对每个分类循环调 GPT，每次出 ~8-10 条并带上已有 english 的 avoid-list，累积去重到 25 条。
