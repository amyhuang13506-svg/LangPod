## 一、背景与设计目标

**产品定位（当前）**：境外生活口语 + 实用口语。核心内容模块：
1. 学习播客（5 遍精听法，三级别）
2. 句型讲解（从播客拆出的高频句型）
3. 出国场景课堂（按国家/场景分类：点单、租房、看病、办事…）
4. YouTube 原声播客（中英双语字幕）
5. 每日任务打卡 + streak

**Onboarding 要达到的三件事**（参考 Speak 思路）：
- **快**：每屏一次点击自动前进，全程 ≤30 秒，无长演示
- **让用户觉得有效**：回答 3 个问题 → 看到一份"为我生成"的计划 + 效果时间线
- **拿到投放需要的数据**：归因来源 + 学习目标（可用于后续内容推荐和推送文案）

## 四、个性化规则表（回答"计划是不是人人一样"）

**现状：不是 AI、没有服务器请求，是本地按 3 个答案模板拼装。** 组合数 = 6 目标 × 3 级别 × 5 时长 = 90 种。

| 用户答案 | 影响什么 |
|----------|----------|
| 级别 | 播客条目难度文案、计划 chip、生成动画第 1 行 |
| 目标 = 刷原声视频 | 第三条目 = YouTube 原声播客 |
| 目标 = 其他 5 个 | 第三条目 = 出国场景课堂，且说明文案随目标变（旅行→机场酒店餐厅 / 留学→校园社交 / 海外生活→租房看病办事 / 工作→职场沟通） |
| 时长 ≥ 20 分钟 | 场景课堂和 YouTube 原声两条都排进计划（5 条） |
| 目标 | 第 90 天里程碑文案（6 套） |

**第 7/21/30/90 天的时间线没有外部数据来源**，是我们自己写的产品承诺。所以文案刻意用体验描述（"能抓到关键词"），不用可证伪的数字（"听力提升 47%"），避免审核风险和用户质疑。

- 推送文案按目标分组（配合现有 push intent 埋点）
