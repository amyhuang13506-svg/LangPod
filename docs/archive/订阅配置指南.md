# Castlingo 订阅配置完整指南

## 总览

订阅功能完成需要 **3 步**：
1. **本地 StoreKit Configuration**（已完成）—— 不依赖网络，离线测试整个购买流程
2. **App Store Connect 创建产品** —— 上架前必须，真机测试和审核都依赖
3. **Xcode Scheme 关联本地 StoreKit 文件** —— 让本地测试用上配置文件

---

## 第一步：本地 StoreKit Configuration（已完成）

文件已创建：`LangPod/Castlingo.storekit`

包含两个订阅产品：

| Product ID | 名称 | 价格 | 周期 | 试用 |
|-----------|------|------|------|------|
| `com.amyhuang.castlingo.pro.yearly` | 年付 | ¥298 | 1 年 | 1 周免费 |
| `com.amyhuang.castlingo.pro.monthly` | 月付 | ¥48 | 1 月 | 无 |

### 关联到 Xcode Scheme（**关键步骤**）

1. Xcode 打开 Castlingo 项目
2. 顶部菜单 **Product → Scheme → Edit Scheme...**（快捷键 ⌘<）
3. 左侧选 **Run**
4. 顶部 tab 选 **Options**
5. 找到 **StoreKit Configuration** 行
6. 下拉选择 **Castlingo.storekit**
7. 点 **Close** 保存

### 验证本地配置生效

1. ⌘R 真机或模拟器启动 app
2. 进入"我的"→ 升级 Pro
3. 看 Paywall：年付下应该显示 **¥298/年** + **7 天免费试用**（这是从 storekit 文件读的，不是 dev mock fallback）
4. 点击"开始免费试用"会弹出**沙盒购买对话框**
5. 选择"购买"→ 进入 ProfileView 验证 Pro 状态（金色"Pro"徽章出现）

如果看到沙盒购买对话框，说明本地 StoreKit 文件已生效。

### 本地测试技巧

打开 Castlingo.storekit 文件后，Xcode 顶部会出现 **Editor** 菜单，里面有：

- **Default Storefront** —— 切换"美国 / 中国 / 日本"等区域，验证 `displayPrice` 本地化
- **Default Locale** —— 切换语言
- **Reset Subscription Renewal Period** —— 加速续费触发（30 秒 = 1 周）
- **Manage Default Storefront → Time Rate** —— 调整试用时长加速

测试要点：
- ✅ 美区切换：年付价格应该自动变成 `$XX/year`（如果你在 storekit 里给了 USD 配置）
- ✅ 续费加速：年付订阅在 30 秒后自动续费，验证状态
- ✅ 试用切换：用户已经用过试用后，再次打开 Paywall 应该看到"无试用"状态

## 第二步：App Store Connect 配置

**前提**：你的开发者账号要满足
- 已签订 Paid Apps Agreement（在 Agreements, Tax, and Banking 里）
- 已完成 Tax Forms（中国大陆开发者要填 W-8BEN）
- 已绑定收款银行账号

否则就算配置了 IAP，购买功能也跑不通。

### 2.1 创建 App（如果还没有）

1. 登录 https://appstoreconnect.apple.com
2. **My Apps → +（左上角）→ New App**
3. 填写：
   - **Platform**: iOS
   - **Name**: Castlingo
   - **Primary Language**: Chinese (Simplified)
   - **Bundle ID**: 选 `com.amyhuang.castlingo`（如果列表里没有，先去 Certificates, IDs & Profiles 创建）
   - **SKU**: castlingo（任意唯一字符串）
   - **User Access**: Full Access
4. **Create**

### 2.2 创建订阅组（Subscription Group）

订阅组的作用：同组内的产品互斥（用户同时只能订阅其中一个），用户可以"升级 / 降级"在年付和月付之间切换。

1. 进入你的 App
2. 左侧菜单 **Monetization → Subscriptions**
3. 点 **+** 创建 Subscription Group
4. **Reference Name**: `Castlingo Pro`
5. **Submit**

进入新建的订阅组：

6. **Subscription Group Localization** 必须填一个：
   - **Display Name**: `Castlingo Pro`
   - **Custom App Name**（留空或填 Castlingo）
7. **Save**

### 2.3 创建年付订阅

在订阅组内点 **+ Create Subscription**：

- **Reference Name**: `Castlingo Pro Yearly`（仅内部用）
- **Product ID**: `com.amyhuang.castlingo.pro.yearly`（**必须和代码里完全一致**）
- **Create**

进入产品详情页：

#### Subscription Duration
- **Subscription Duration**: 1 Year

#### Subscription Prices
- 点 **+ Add Pricing**
- **Starting Price (Base Country)**: 中国 ¥298 (CNY)
- 系统会自动按汇率换算其他国家价格，可以一眼扫一下，调整不合理的（比如美区给 $39.99）
- **Save**

#### Localizations
点 **+ Add Localization**：

**Chinese (Simplified)**:
- **Display Name**: `年付`
- **Description**: `Castlingo Pro 年度订阅，解锁全部高级功能：每日无限播客、5 遍循环播放、双语字幕、生词卡片、词义配对、连词成句。`

**English (US)**:（建议补一份英文，提高过审通过率）
- **Display Name**: `Yearly`
- **Description**: `Castlingo Pro Yearly Subscription. Unlock all premium features: unlimited daily podcasts, 5-round playback, bilingual subtitles, vocabulary cards, word matching, sentence building.`

**Save**

#### Introductory Offers（**关键 - 这才是 paywall 显示"7 天免费试用"的源头**）

1. 滚到下方 **Introductory Offers** section
2. 点 **+ Create Introductory Offer**
3. **Countries or Regions**: All Countries（或者只选中国先）
4. **Start Date**: Now
5. **End Date**: No End Date
6. **Offer Type**: **Free Trial**
7. **Duration**: **1 Week**
8. **Save**

**注意**：苹果只允许这几个时长：3 days / 1 week / 2 weeks / 1 month / 2 months / 3 months / 6 months。代码里的 `displayFromPeriod` 会把 `1 week` 显示为"7 天"。

#### App Store Promotion（可选，先跳过）

#### Review Information（审核必填）
- **Screenshot**: 上传一张 paywall 截图（任意尺寸，640×920 以上）
- **Review Notes**: `年度订阅，¥298/年，1 周免费试用。在我的页面 → 升级 Pro 触发购买流程。`

**Save**

### 2.4 创建月付订阅

回到 Subscription Group，再点 **+ Create Subscription**：

- **Reference Name**: `Castlingo Pro Monthly`
- **Product ID**: `com.amyhuang.castlingo.pro.monthly`
- **Create**

#### Subscription Duration
- **1 Month**

#### Subscription Prices
- Base: ¥48 (CNY)
- 系统自动换算其他国家

#### Localizations

**Chinese (Simplified)**:
- **Display Name**: `月付`
- **Description**: `Castlingo Pro 月度订阅，解锁全部高级功能。`

**English (US)**:
- **Display Name**: `Monthly`
- **Description**: `Castlingo Pro Monthly Subscription. All premium features unlocked.`

#### Introductory Offers
**不要配置！** 这是产品策略：月付永远没有免费试用，鼓励用户选年付。

代码里 `effectiveMonthlyTrial` 永远返回 `nil`，即使你不小心在 App Store Connect 配了月付试用，UI 也不会显示。

#### Review Information
- **Screenshot**: 同年付，可复用
- **Review Notes**: `月度订阅，¥48/月，无试用。`

**Save**

### 2.5 触发产品状态变更

刚创建的订阅默认状态是 **Missing Metadata** 或 **Ready to Submit**。要让真机能拉到产品：

1. 确认两个订阅都达到 **Ready to Submit** 状态（所有必填项都填了）
2. **不需要单独提交** —— 提交 app 审核时自动一起带上
3. **本地真机测试**：用 Sandbox 账号即可，不需要正式审核

### 2.6 创建 Sandbox 测试账号

1. App Store Connect 左侧 **Users and Access → Sandbox → Testers**
2. **+ Add Tester**
3. 用一个**没有用过的**邮箱（推荐用 yourname+sandbox@gmail.com 这种 + 别名）
4. 填一个简单的密码（之后每次买都要输）
5. **Country/Region**: China
6. **Invite**

**关键**：在 iPhone 上**先登出真实 Apple ID 的 App Store / iTunes 账号**，然后：
- 设置 → 开发者 → Sandbox Apple Account → Sign In
- 使用 sandbox 账号登录（不需要登出主 Apple ID）

之后在 Castlingo 里点购买，系统会用 Sandbox 账号弹窗，不收真钱。

## 常见坑

**坑 1: Product ID 大小写不对**  
代码里写的是小写 `com.amyhuang.castlingo.pro.yearly`，App Store Connect 里也必须严格小写一致。一字之差产品就拉不到。

**坑 2: 没签 Paid Apps Agreement**  
免费 app 配 IAP 的最常见踩坑。Agreements, Tax, and Banking 里要有"Paid Apps"且状态绿色 Active。

**坑 3: Sandbox 账号未隔离**  
用主 Apple ID 试买会扣真钱。一定要在 设置 → 开发者 → Sandbox 里登 sandbox 账号。

**坑 4: Bundle ID 不匹配**  
Xcode 项目的 Bundle ID 必须和 App Store Connect 创建 app 时选的 Bundle ID 完全一致。否则真机虽然能装，但购买时拉不到产品。

**坑 5: 试用配置没生效**  
如果在 App Store Connect 配了 1 Week Free Trial 但 paywall 还显示"无试用"，检查：
- Sandbox 账号是不是已经"用过"了试用（每个 Apple ID 每个订阅组只能试用一次）
- 换个 sandbox 账号试

## 上架前最后一步

两个订阅都 Ready to Submit 之后：
1. 进 app 主页 → **App Information** 把所有必填项填完
2. **Pricing and Availability** 设置 app 价格（免费）和上架国家
3. **App Privacy** 完善隐私清单（要加订阅条目）
4. **+ Version** 创建第一个版本号 1.0.0
5. 上传截图、描述文案
6. 在 1.0.0 的 **In-App Purchases** 里**勾选**两个订阅一起提交
7. **Submit for Review**

整个流程从配置到提交大概 1-2 小时（不含等审核）。
