# 会员权限系统修正方案

## 一、当前问题

### 问题 1：每日集数限制形同虚设
- `dailyEpisodesPlayed` 只在集数**播完**时 +1，用户可以无限**开始**新集
- 大量播放入口绕过了 gate 检查：
  - PlayerView 的上一集/下一集按钮
  - AllEpisodesView 的集列表
  - StatsView 的播放历史
  - HomeView 的快进按钮
  - 完成页的"下一集"按钮

### 问题 2：免费用户感知不到第 5 遍
- 免费用户只播 4 遍，但 UI 上没有任何提示"还有第 5 遍是 Pro 专属"
- 用户不知道自己少了什么，没有升级动力

### 问题 3：Paywall 没有提到 5 遍
- 功能列表里没有"完整 5 遍播放"这个卖点

### 问题 4：词汇本整体锁死太粗暴
- 用户完全看不到自己的词汇，体验割裂
- 应该让用户能看词汇、每日免费练一轮

### 问题 5：苹果审核合规
- 当前 Paywall 基本合规（有价格、试用期、恢复购买、隐私/条款链接）
- 但需确保：试用期文案明确写"3 天后自动续费 ¥XX"，不能含糊

## 二、修正方案

### 2.1 每日集数限制 — 改为「开始播放时计数」

**逻辑**：
- 计数时机从"播完"改为"开始播放新集"
- 在 AudioPlayer 层做统一拦截，而非在每个 View 里加 guard
- 新增 `AudioPlayer.onPlayBlocked` 回调，View 层收到后弹 Paywall

**具体改动**：

| 文件 | 改动 |
|------|------|
| `AudioPlayer.swift` | 新增 `playGate: ((Episode) -> Bool)?` 回调。`playEpisode()` 调用前先检查 gate，不通过则调 `onPlayBlocked` 并 return |
| `DataStore.swift` | `recordDailyPlay()` 改为在 gate 检查时调用（开始播放时），不再在 completeEpisode 里调用 |
| `HomeView.swift` | 移除分散的 `canPlayOrShowPaywall()` guard，改为监听 `audioPlayer.onPlayBlocked` 弹 Paywall |
| `PlayerView.swift` | 监听 `audioPlayer.onPlayBlocked` 弹 Paywall |
| `AllEpisodesView.swift` | 监听 `audioPlayer.onPlayBlocked` 弹 Paywall |
| `StatsView.swift` | 监听 `audioPlayer.onPlayBlocked` 弹 Paywall |
| `LangPodApp.swift` | 在根层设置 `audioPlayer.playGate`，统一判断 isPro + dailyCount |

**好处**：一个入口拦截所有播放，不再有遗漏。

**额外规则**：
- 重复播放同一集不计数（只有切换到新集才 +1）
- 上一集/重播不计数

### 2.2 第 5 遍感知 — 播放页添加 Pro 提示

**方案**：免费用户在第 4 遍（翻译遍）播完后，不直接跳 finished，而是：

1. 显示一个**内嵌提示卡片**（不是弹窗），位于播放页遍数标签下方：
   ```
   ┌─────────────────────────────┐
   │  🔒 Pro 专属：第 5 遍英语原音  │
   │  再听一遍加深记忆              │
   │  [升级 Pro]                   │
   └─────────────────────────────┘
   ```
2. 提示卡片显示 3 秒后自动进入 finished 状态
3. 用户点"升级 Pro"→ 弹 Paywall

**改动文件**：
- `PlayerView.swift`：在 phase badge 下方条件渲染提示卡片
- `AudioPlayer.swift`：新增 `.proUpsell` phase（在 translationRound 和 finished 之间），3 秒后自动 advance

### 2.3 Paywall 功能列表更新

当前列表：
```
无限播客集数
词义配对 + 连词成句练习
完整词汇本，无限保存
离线缓存 + 倍速播放
Streak 冻结卡
```

改为：
```
每日无限集数（免费每日 2 集）
完整 5 遍播放 + 双语字幕
无限词义配对 + 连词成句
完整词汇本 + 记忆追踪
离线缓存 + 倍速播放
```

去掉 Streak 冻结卡（MVP 不做）。

**改动文件**：`PaywallView.swift`

### 2.4 词汇本 — 开放浏览，限制练习次数

**改为**：
- 词汇本页面**完全开放**，免费用户可以浏览所有已保存的词汇
- 保存词汇功能**开放**（完成页"保存到我的词汇"不限制）
- **每日免费 1 轮**词义配对 + **每日免费 1 轮**连词成句
- 用完后点击按钮 → 弹 Paywall

**具体改动**：

| 文件 | 改动 |
|------|------|
| `VocabularyView.swift` | 移除整页锁定。恢复原始 UI，练习按钮加每日次数检查 |
| `VocabularyStore.swift` | 新增 `dailyMatchPlayed` / `dailySentencePlayed`（UserDefaults，每日重置） |
| `WordMatchView.swift` | 完成一轮后标记 `dailyMatchPlayed = true` |
| `FeynmanChallengeView.swift` | 完成一轮后标记 `dailySentencePlayed = true` |
| `EpisodeCompleteView.swift` | "保存到我的词汇"按钮不限制（移除 Pro gate） |

### 2.5 苹果审核合规检查

当前基本合规，需微调：

| 项目 | 当前状态 | 需改动 |
|------|---------|--------|
| 恢复购买按钮 | 已有 | 无需改 |
| 试用期说明 | "3天免费试用" + 日期 | 补充"试用结束后按 ¥XX/年 自动续费" |
| 隐私政策/条款链接 | 已有 | 无需改 |
| 取消说明 | "可随时取消" | 补充"在 App Store 设置中管理订阅" |
| 价格展示 | 有 | 确保价格是最醒目的元素 |

**改动文件**：`PaywallView.swift` — 修改底部合规文案
