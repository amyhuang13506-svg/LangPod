# 词汇小课堂 · 内容生产标准（Content Production Runbook）

> 本文档是词汇小课堂内容的**输出标准 + 跑批操作手册**，自包含，可在新 session 直接照此执行。
> 已验收基准：`lesson_us_otc_meds`（在 CVS 买非处方药）、`lesson_us_supermarket`（在 Target 购物）、`lesson_us_chipotle`（在 Chipotle 排队点餐）——内容和图片均已确认 OK，新内容以这三个为质量对照。

## 一、产品定位（选题的唯一标准）

每个课堂是一个**"具体到能想起画面"的英语国家真实生活时刻**，不是教材章节。
- ✅ 在 CVS 买非处方药 / 被警察拦下 / Chipotle 排队一路选配料
- ❌ 机场 / 银行 / 购物（教材式泛场景）
词表偏向"教科书不教但当场就要用"的词：Tylenol、copay、self-checkout、guac is extra。品牌词、当地俚语是加分项不是问题。

## 二、目录矩阵（内容从哪来）

全部课堂定义在 **`pipeline/lesson_catalog.py`**（人工定稿，不交给 GPT 编目录）：
- `COUNTRIES`：6 国（us/uk/au/ca/nz/sg），含口音代码、拼写规范、本地化上下文
- `SCENE_TEMPLATES`：~16 个共享场景模板 × 每国本地品牌锚点（买药 = CVS/Boots/Chemist Warehouse/Shoppers/Guardian）
- `COUNTRY_EXTRAS`：每国 3-6 个特色课堂（美国被警察拦下、英国 Pub、新加坡 Singlish…）
- 展开共 **125 个课堂**（us 22 / uk 21 / au 21 / ca 20 / nz 20 / sg 21）
- 免费策略：`FREE_SLUGS`（customs/supermarket/otc_meds/bank_account/campus 每国免费），其余 Pro

新增课堂 = 改 lesson_catalog.py（模板加国家 or COUNTRY_EXTRAS 加条目），id 约定 `lesson_{country}_{slug}`。

## 三、单课堂内容标准

### 结构
- 3-5 个分区（zone），每分区一张场景图
- 每分区：5-8 个 **hotspots**（图上可标注的实物名词）+ 3-6 个 **extra_words**（画不出来的动词短语/抽象表达）
- 全课堂 20-35 词 + 5 个常用句型 + 2-3 条文化贴士

### 词条字段（每个词必须齐全）
| 字段 | 标准 |
|------|------|
| word | 小写（专有名词除外）；hotspot 必须是"插画里画得出来且视觉可辨认的实物" |
| phonetic | IPA 带斜杠 `/ˈpæspɔːrt/` |
| translation_zh | 1-2 个自然口语化中文释义，不要词典腔 |
| example | 该场景下真人会说的一句话，**必须包含该词**，**≤12 个单词**（连词成句玩法约束） |
| example_zh | 自然中文翻译 |
| difficulty | easy/medium/hard 按真实使用频率；品牌词/当地词标 medium/hard |

硬规则：同一课堂内单词不重复；分区内按易→难排序；英式国家用英式拼写和当地叫法（trolley/petrol/chemist）；禁止政治宗教敏感内容。

### 句型（sentences，5 条）
该场景下用户真的会**说出口或听到**的句子，口语语域不要教材腔。（句型 tab 按"国家+场景"直接展示这批句子，质量要求同正文。）

### 文化贴士（culture_tips_zh，2-3 条）
中文"生存知识"：给多少小费、什么不能做、当地人实际怎么说。禁止百科式介绍。
- ✅ "被 pull over 双手放方向盘上，别下车别翻包"
- ❌ "CVS 成立于 1963 年，是美国最大连锁药店"

## 四、图片标准

- **模型：`gpt-image-1`**（不要用 DALL-E 3——它会生成乱拼的假英文单词，对英语学习 app 是致命伤）
- 尺寸 1536×1024 横幅，quality medium
- 风格模板（脚本里已固化）：扁平矢量插画、暖色奶油底、轮廓干净、类似设计感 app 的插画
- 文字规则：图内文字尽量少；**出现的文字必须是拼写正确的真实英文**（PHARMACY、SALE）；**禁止真实品牌 logo/商标**（场景只画"美式药店"的典型环境）
- 每张图必须让该分区全部 hotspot 物品清晰可见、互不遮挡、分散布局
- 封面 = 第一分区图复制（零成本）

## 五、坐标标注标准

- 视觉模型：GPT-4o（经 api.v3.cm），输入图 + 单词列表 → 每词归一化中心坐标 (x, y)
- 坐标 clamp 到 x∈[0.04, 0.96]、y∈[0.06, 0.94]
- 闭环规则（脚本自动执行）：有词没识别到 → 换构图措辞重新生成整图（最多 2 次）→ 仍失败的词**自动降级进 extra_words**（绝不硬标）
- 分区合格线：存活 hotspot ≥ 3 个
- 实测基准：13 张图 11 张首次全中，重试后全中

## 六、发音音频标准

- **模型：ElevenLabs `eleven_flash_v2_5`**（半价，短句质量足够；key 在 config.py 的 `ELEVENLABS_LESSON_API_KEY`）
- 覆盖：每个单词 + 每条例句 + 每个句型，各一个独立 mp3
- 声音按国家（config.py `ELEVENLABS_LESSON_VOICES`）：us=Bella(美) / uk=Alice(英) / au=Charlie(澳) / ca=Sarah(美) / nz=Charlie(澳) / sg=Daniel(英)
- 参数：mp3_44100_64，stability 0.6，speed 0.9
- 中文不合成（界面文字）；App 端拿不到音频自动回落系统 TTS

## 七、跑批操作（在 pipeline/ 目录执行）

```bash
cd /Users/mac/Desktop/LangPod/pipeline

# 0. 自检目录矩阵（应输出 125 个课堂的分国统计）
python3 lesson_catalog.py

# 1. 生成课堂内容（GPT-4o，自动质检+重试，幂等：已存在跳过）
python3 generate_lessons.py --country us        # 按国家
python3 generate_lessons.py --lesson lesson_uk_pub   # 或单课堂
python3 generate_lessons.py                     # 全量

# 2. 生成场景图 + 自动标注（gpt-image-1 + GPT-4o vision，幂等）
python3 generate_lesson_images.py --country us
#    产出人工审阅页：output/lessons/review_us.html（浏览器打开肉眼扫标注）

# 3. 合成发音音频（ElevenLabs，幂等：已有 mp3 跳过）
python3 generate_lesson_audio.py --country us

# 4. 上传 OSS + 重建索引（图片/音频/JSON，重写 URL，重建 index.json + countries.json）
python3 upload_lessons.py --country us
```

推荐节奏：**按国家跑**（us → uk → au → ca → nz → sg），每国跑完先开 review_{cc}.html 抽查再上传；中断随时重跑（全链路幂等）。

## 八、质检与验收

**脚本自动质检**（generate_lessons.py 内置，不达标自动重试）：JSON 结构、zone id 对齐、hotspot 5-8 个、词条字段齐全、例句含目标词、难度合法、跨分区无重复词。

**人工抽查（review_{country}.html，每国跑完必看）**：
1. 标注点是否落在物体上（个别偏移可接受，整体错位则该课堂重跑图）
2. 图内文字拼写是否正确、有无品牌 logo
3. 词表是否够"当地真实"（英国是否用了 trolley/chemist，新加坡例句是否有本地语境）
4. 文化贴士是否"生存知识"而非百科
5. 抽听 2-3 条音频（口音对不对、有没有截断）

**上架后真机验证**：切到该国家 → 进新课堂 → 图片加载、标注可点、发音正常、加词入库。
