## 产品定位

句型 tab = 口语表达库。4 个大组 × 24 个功能分类，共 227 条口语表达（不按国家、不分难度，实用频率排序，国家差异写进每条的 `country_note_zh`）。

**交互形态（已定稿）**：每个分类一个区块（标题 + 查看更多 + 单行横滑卡片）；**每张卡 = 一条表达，封面 = 它自己的场景插画**；点卡进翻页详情（场景插画 + App 叠加对话气泡 + 讲解 + 4 例句），左右滑切表达。

## 主任务：跑剩余 218 张场景插画

```bash
cd ~/Desktop/LangPod/pipeline

# 1) 全量跑（幂等：已有图跳过，greetings 会自动 skip）
python3 -u generate_expression_scenes.py            # 全部分类
# 或单分类试跑：python3 -u generate_expression_scenes.py thanks

# 2) 上传（重跑安全；场景图 URL 会写回分类 JSON + index 封面自动换成第一条场景图）
python3 upload_expressions.py
```

- 每张 ~30-60 秒，218 张预计 2-3 小时；**建议后台跑 + 定期看输出**
- 单张成本 ~¥0.5（gpt-image-1 1536x1024 medium），全量约 **¥110**
- 网络失败：直接重跑同一命令（skip-existing 续跑）
- 失败张数不为 0 时重跑一遍，仍失败的看报错（多为代理超时）

### 场景图质量标准（prompt 已内置，抽查时按此验收）

1. **恰好两个人**：A 在左 1/3、B 在右 1/3，正在对话，外观有区分（发色/衣服颜色）
2. **脸在画面上 1/3**，画面底部保持简单 —— App 会把对话气泡叠加在**底部**，绝不能挡脸
3. **图内零文字**：无字母/数字/招牌/logo/自带气泡（台词由 App 渲染，保证零拼写错误）
4. 扁平插画风、暖色、cream 底（与词汇小课堂封面同风格）
5. 人物姿态/表情要贴合该条对话内容（prompt 会带入 setup_zh + 对话台词）

### 文件与命名规则（防串图，勿改）

- 输出：`output/expressions/scenes/{分类id}/{slug}_{内容hash8}.png`
- hash = md5(setup_zh + 全部对话英文)[:8] —— **场景文字一变，旧图自动作废重生成**（音频串戏事故的同款教训）
- 写回分类 JSON 的 `scene.image`（相对路径），upload 时换成 OSS URL 并压缩为 .jpg

## 次任务：例句扩充收尾（若主 session 没跑完）

```bash
# 幂等：已有 4 例句的表达跳过；音频只补缺的 ex2/ex3
python3 -u generate_expressions.py --augment-examples
python3 upload_expressions.py
```

例句标准：≤12 词、真实口语语域（缩写 OK）、与已有例句话题不重复、中文翻译自然不直译。

## 收尾：音频核对（必须）

```bash
# 断点续跑；新增例句音频（ex2/ex3）+ 未验过的都会补验
python3 verify_expression_audio.py
```

- 输出 `=== total N verified, M problems ===`，**M 必须为 0**
- 出现 MISMATCH：删掉该条 JSON 里对应的 audio 字段 + 本地 mp3，重跑 `generate_expressions.py --augment-examples`（或对应生成命令）→ 再 upload → 再 verify
- checkpoint 在 `output/expressions/verify_ckpt.json`，想全新跑删掉它

## App 端（无需改动）

- App 读 `expressions/index.json`（network-first），场景图上传后**重进句型 tab 即生效**，不用发版
